Depending on the use case, you can either use this library as-is (which applies some [default options](./src/internal/default/options.ts)), or customize it to your liking. If you choose to do so, you can pass an object to this action containing your own options to be applied.

Keep in mind that these options are applied to the single DOM element you add Svelte Reveal to. For global and more in-depth settings, refer to the [API](/reference/api) section.

### `disable`

**type**: `boolean` <br/>
**default**: `false`

When set to false, the transition is disabled for the target element.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ disable: false }}>Hello world</h1>
```

### `root`

**type**: `Element | Document | null` <br/>
**default**: `null`

The [root](https://developer.mozilla.org/en-US/docs/Web/API/IntersectionObserver/root) element used by the Intersection Observer.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ root: null }}>Hello world</h1>
```

### `rootMargin`

**type**: `string`<br/>
**default**: `"0px 0px 0px 0px"`

The [root margin](https://developer.mozilla.org/en-US/docs/Web/API/IntersectionObserver/rootMargin) property of the Intersection Observer.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ rootMargin: '0px 0px 0px 0px' }}>Hello world</h1>
```

### `threshold`

**type**: `number`<br/>
**default**: `0.6`

The [threshold](https://developer.mozilla.org/en-US/docs/Web/API/IntersectionObserver/thresholds) (in percentage from `0.0` to `1.0`) property used by the Intersection Observer to know when its target element is considered visible.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ threshold: 0.6 }}>Hello world</h1>
```

### `preset`

**type:** <code>["fade"](#fade) | ["slide"](#slide) | ["fly"](#fly) | ["spin"](#spin) | ["blur"](#blur) | ["scale"](#scale)</code><br/>
**default**: `"fade"`

The transition preset that should be applied. Check out [presets](#presets) for more info.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ preset: 'fade' }}>Hello world</h1>
```

### `reset`

**type**: `boolean`<br/>
**default**: `false`

<Aside type="caution" title="Be careful">
  Overusing this option prevents the Intersection Observer to stop observing the target node. Performance is therefore
  not guaranteed when many elements have `reset` set to `true`.
</Aside>

When set to `true`, the node transitions out when out of view, and is revealed again when back in view.

```svelte
<script lang="ts">
  import { reveal } from 'svelte-reveal';
</script>

<h1 use:reveal={{ reset: false }}>Hello world</h1>
```

## Presets

<Aside type="caution">All presets have the `"fade"` preset backed in</Aside>

Presets are sets of options with predefined values, packaged under a name to achieve a certain transition effect. The following table shows the presets that come bundled with Svelte Reveal and which options they map to.

### `fade`

The element fades in gracefully.

#### Mapped options

```typescript
{
  opacity: 0;
}
```

#### Resulting CSS

<Tabs>
<TabItem label="Start">

```css
opacity: 0;
```

</TabItem>
<TabItem label="End">
  
```css 
opacity: 1;
```

</TabItem>
</Tabs>

### `fly`

The element fades in and moves along a translation on the y-axis.

#### Mapped options

```typescript

  y: -20;

```


</TabItem>
</Tabs>
